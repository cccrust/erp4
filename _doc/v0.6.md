# ERP4 v0.6 — Web 網站 (Axum + React)

在 v0.5 CLI 使用者/權限系統基礎上，建立 Web 管理介面。

## 總體架構

```
erp4 web [--port 8080] [--host 127.0.0.1]
```

- 單一二進位，既存 CLI 模式不動
- Web 模式讀取同一 SQLite 資料庫（`ERP4_DB`）
- 模型層完整重用，API handler 直接呼叫 model 函式
- React SPA 編譯後由 Axum 提供靜態檔案服務
- 認證使用 JWT（重用 v0.5 users + sessions 邏輯）

## API 路由

所有 API 路徑以 `/api/` 前綴，認證使用 `Authorization: Bearer <JWT>`。

### 認證（無需 token）

| 方法 | 路徑 | 請求 | 回應 |
|------|------|------|------|
| POST | `/api/login` | `{ username, password }` | `{ token, user: { id, username, display_name, role } }` |
| GET | `/api/me` | — | `{ id, username, display_name, role }` |

### 儀表板

| 方法 | 路徑 | 回應 |
|------|------|------|
| GET | `/api/dashboard` | `{ customer_count, product_count, low_stock_count, pending_orders, pending_pos, overdue_invoices_total, recent_orders: [...] }` |

### 客戶

| 方法 | 路徑 | 參數 |
|------|------|------|
| GET | `/api/customers` | `?search=&page=&page_size=` |
| GET | `/api/customers/:id` | — |
| POST | `/api/customers` | `{ name, email?, phone?, address? }` |
| PUT | `/api/customers/:id` | `{ name?, email?, phone?, address? }` |
| DELETE | `/api/customers/:id` | — |

產品、供應商比照客戶模式。

### 訂單

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/orders` | `?status=&customer_id=&page=&page_size=` |
| GET | `/api/orders/:id` | 含明細 |
| POST | `/api/orders` | `{ customer_id, notes? }` |
| POST | `/api/orders/:id/status` | `{ status }` |
| DELETE | `/api/orders/:id` | — |
| GET | `/api/orders/:id/items` | 明細列表 |
| POST | `/api/orders/:id/items` | `{ product_id, quantity, unit_price? }` |

採購單比照訂單模式，路徑為 `/api/purchase-orders`。

### 發票

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/invoices` | `?status=&customer_id=&page=&page_size=` |
| GET | `/api/invoices/:id` | — |
| POST | `/api/invoices` | `{ invoice_number?, order_id?, customer_id, due_date, amount, notes? }` |
| POST | `/api/invoices/:id/status` | `{ status }` |
| DELETE | `/api/invoices/:id` | — |

### 報表

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/reports/sales` | `?from=YYYY-MM-DD&to=YYYY-MM-DD` |
| GET | `/api/reports/inventory` | — |
| GET | `/api/reports/aging` | — |

## 技術選型

| 層 | 技術 | 說明 |
|---|------|------|
| HTTP | `axum` | async, tokio 生態 |
| Runtime | `tokio` (full) | async 執行器 |
| Auth | `jsonwebtoken` + `argon2` | 無狀態 JWT 認證 |
| Serialization | `serde` + `serde_json` | JSON 序列化 |
| Frontend | React + TypeScript + Vite | SPA |
| 路由 | `react-router-dom` v7 | 前端路由 |
| UI | Tailwind CSS | utility-first CSS |
| HTTP client | 原生 `fetch` | 不引入 axios |
| Build | `cargo build --release` + `npm run build` | 兩步驟 |

## 認證流程

```
LoginPage ──POST /api/login──→ Axum 驗證 username + argon2(password)
                                ├─ 成功 → 簽發 JWT (sub=user_id, exp=24h)
                                │         回傳 { token, user }
                                └─ 失敗 → 401 { error: "帳號或密碼錯誤" }

Client 儲存 JWT 於 localStorage，每次 API 請求帶上
Authorization: Bearer <token>

Axum middleware 驗證 JWT：
  ├─ 有效 → 注入 AuthUser extractor
  └─ 無效/過期 → 401 { error: "未授權" }
```

## 開發流程

```bash
# 終端 1：啟動 Rust API server（hot reload 用 cargo-watch）
ERP4_DB=erp4-dev.db cargo run -- web --dev

# 終端 2：啟動 Vite dev server（proxy API 至 8080）
cd web && npm run dev
```

Vite dev server (port 5173) 會 proxy `/api/*` 請求至 Axum (port 8080)，解決 CORS 問題。

## 正式建置

```bash
# 1. 建置前端
cd web && npm ci && npm run build   # 輸出至 web/dist

# 2. 建置 Rust 二進位
cargo build --release               # Axum 嵌入 web/dist 靜態資源

# 3. 執行
./target/release/erp4 web           # 監聽 127.0.0.1:8080
```
