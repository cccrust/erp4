# ERP4 v0.5 — CLI 強化

補齊 v0.4 未實作項目 + 使用者/權限系統。純 CLI 變更，不觸及 Web 層。

## 新增功能

### ✅ 1. CSV 資料匯入（from v0.4）

新增 `import` 子指令，支援從 CSV 檔案批次建立客戶、產品、供應商。

```
erp4 customer import customers.csv
erp4 product  import products.csv
erp4 supplier import suppliers.csv
```

**CSV 欄位對應（首列為標題列）：**

| 實體 | 必要欄位 | 可選欄位 |
|------|---------|---------|
| customer | `name` | `email`, `phone`, `address` |
| product | `name`, `sku`, `price` | `stock`, `description` |
| supplier | `name` | `contact_person`, `email`, `phone`, `address` |

- 支援 UTF-8 BOM
- 逐列處理，單列錯誤不中斷
- 顯示統計：`已匯入 47 筆，跳過 3 筆錯誤`
- 自訂輕量 CSV parser（不引入 csv crate）

### ✅ 2. JSON 輸出格式（from v0.4）

所有 `list`、`get`、`report` 指令支援 `--format json`。

```
erp4 customer list --format json
erp4 product  get  1  --format json
erp4 order    list --status confirmed --format json
erp4 report   sales --from 2026-01-01 --to 2026-06-30 --format json
```

- `list` 輸出 JSON 陣列
- `get` 輸出單一 JSON 物件
- 需新增 `serde_json` 相依
- 所有資料結構 derive `Serialize`

### ✅ 3. 匯出全部資料 (`erp4 export`)

```
erp4 export --format json > backup.json
erp4 export --format csv > backup.csv
```

- JSON：`{ "customers": [...], "products": [...], "suppliers": [...], "orders": [...], "purchase_orders": [...], "invoices": [...] }`
- CSV：多個區塊以註解行 `# <table_name>` 分隔

### ✅ 4. 使用者管理（新）

新增 `users` 資料表，以及 CLI 使用者管理指令。

```
erp4 user create <username> [--password] [--display-name] [--role]
erp4 user list
erp4 user get <id>
erp4 user update <id> [--password] [--display-name] [--role]
erp4 user delete <id>
```

- 密碼以 **argon2** 雜湊儲存
- 角色：`admin` / `viewer`（供 v0.6 Web 使用）
- 首次 `erp4 init` 可自動建立預設管理員帳號

### ✅ 5. Session 管理（新）

新增 `sessions` 資料表，以及 CLI 登入/登出/查詢指令（主要為驗證與偵錯用途）。

```
erp4 session login <username> [--password]
erp4 session logout <session-id>
erp4 session list
```

- Session 以 UUID 為 token
- 有效期限可設定（預設 24 小時）

## 資料庫變更

```sql
CREATE TABLE IF NOT EXISTS users (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    username     TEXT NOT NULL UNIQUE,
    password     TEXT NOT NULL,        -- argon2 hash
    display_name TEXT NOT NULL,
    role         TEXT NOT NULL DEFAULT 'admin',
    created_at   TEXT NOT NULL,
    updated_at   TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS sessions (
    id          TEXT PRIMARY KEY,      -- UUID
    user_id     INTEGER NOT NULL REFERENCES users(id),
    expires_at  TEXT NOT NULL,
    created_at  TEXT NOT NULL
);
```
